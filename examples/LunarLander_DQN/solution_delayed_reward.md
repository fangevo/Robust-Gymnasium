# Delayed Reward Structure Analysis – DQN on LunarLander-v3

## 1. Problem Formulation

In reinforcement learning the **credit assignment problem** asks: *which past
actions are responsible for the rewards the agent receives?*  When rewards are
**dense** (delivered at every time-step) the signal-to-noise ratio is high and
temporal-difference (TD) methods can propagate value information efficiently.
When rewards are **delayed** or **sparse**, the agent must bridge a potentially
long temporal gap between an action and its consequence, making learning
dramatically harder.

Formally, given a standard MDP \((S, A, P, R, \gamma)\) with episode length
\(T\), we consider a family of *reward delivery functions*
\(\tilde{R}_k\) parameterised by delay \(k\):

| Mode | Delivered reward at step \(t\) |
|------|-------------------------------|
| **Baseline** (dense, \(k{=}0\)) | \(\tilde{r}_t = r_t\) |
| **Dense (extra shaping)** | \(\tilde{r}_t = r_t + \Phi(s_t)\), additional potential-based bonus |
| **Medium delay** (\(k{=}K\)) | \(\tilde{r}_t = \sum_{i=t-K+1}^{t} r_i\) every \(K\) steps; 0 otherwise |
| **Sparse** (\(k{=}T\)) | \(\tilde{r}_t = 0\) for \(t < T\); \(\tilde{r}_T = \sum_{i=1}^{T} r_i\) |

The total undiscounted return is identical across all modes—only the *timing*
of information delivery changes.  This setup isolates the pure effect of reward
delay on learning dynamics without altering the underlying task.


## 2. Environment & Algorithm

### LunarLander-v3 (Discrete)

- **State** (8-dim): position, velocity, angle, angular velocity, leg contacts.
- **Actions** (4): noop, left engine, main engine, right engine.
- **Default reward**: potential-based shaping \(-100\sqrt{x^2+y^2}\),
  velocity/angle penalties, fuel cost, ±100 terminal bonus.
- Episode terminates on landing, crash, or the 1000-step time limit.

### DQN Configuration

| Parameter | Value |
|-----------|-------|
| Hidden layers | 2 × 128 (ReLU) |
| Learning rate | 5 × 10⁻⁴ (Adam) |
| Replay buffer | 100 000 |
| Batch size | 64 |
| Target update | Polyak τ = 10⁻³ |
| ε-greedy | 1.0 → 0.01, decay 0.995/ep |
| γ | 0.99 |
| Train every | 4 steps |
| Training episodes | 500 (main) / 300 (robustness sweep) |


## 3. Experimental Design

### 3.1 Reward Configurations

Five reward delivery modes are compared:

1. **Baseline** – The unmodified LunarLander-v3 reward.
2. **Dense (extra shaping)** – Baseline + per-step bonus:
   \[
   \text{bonus}_t = 0.2(l_1 + l_2) - 0.3|x| - 0.2|\theta|
   \]
   where \(l_1, l_2\) are leg-contact indicators, encouraging upright,
   centred descent.
3. **Medium Delay (K=10)** – Rewards accumulated over 10 steps, then
   delivered as a single sum.
4. **Medium Delay (K=20)** – Same, with a 20-step accumulation window.
5. **Sparse (Terminal)** – The entire episode reward is delivered at
   termination; zero signal at all intermediate steps.

### 3.2 Robustness Sweep

To map the full degradation curve, DQN is also trained with medium-delay
delivery for \(K \in \{1, 5, 10, 15, 20, 30, 50, 100\}\) plus a sparse
condition (\(K = T\)).

### 3.3 Implementation: DelayedRewardWrapper

A lightweight wrapper sits between the agent and the Gymnasium environment.
It intercepts the reward signal and applies the delay logic while passing
all other quantities (states, termination flags) through unchanged.  The
original reward is always stored in `info["original_reward"]` so that
training curves use the true environment reward for fair comparison.

```
Agent ↔ DelayedRewardWrapper ↔ robust_gymnasium.make("LunarLander-v3")
```

### 3.4 Metrics Collected

| Metric | Granularity | Purpose |
|--------|-------------|---------|
| Original episode reward | Per episode | Learning curve |
| Delivered reward | Per step | Reward timeline |
| Q(s, a) | Per step | Value prediction accuracy |
| \|TD-error\| | Per step | Credit assignment difficulty |
| Return-to-go variance | Per timestep | Estimation uncertainty |
| Final performance vs delay | Per delay value | Robustness profile |


## 4. Visualizations

Six complementary plots are generated by `visualize_delayed_reward.py`.

### Plot 1 – Reward Timeline (timestep vs reward)

Shows the **delivered** reward signal (bars) alongside the **original**
reward (gray line) for one evaluation episode under each mode.  Baseline
and dense show reward at every step; medium-delay shows periodic bursts;
sparse collapses all reward into a single terminal spike.

**Insight**: The temporal sparsity pattern directly illustrates the credit
assignment challenge—as delay increases, the per-step information entropy
of the reward signal drops to zero for most time-steps.

### Plot 2 – Return Variance (timestep vs Var(R_t))

Computes Var(return-to-go) at each timestep across 20 evaluation episodes.
Higher variance at early timesteps means the agent faces more uncertainty
about future outcomes.

**Insight**: Sparse reward typically shows uniformly high variance because
the agent receives no intermediate feedback to reduce uncertainty.  Dense
reward yields a monotonically decreasing variance curve as the episode
progresses and the remaining return becomes more predictable.

### Plot 3 – Value Prediction (timestep vs Q(s,a))

Plots the mean Q-value predicted by the trained agent at each timestep.
A well-trained agent produces Q-values that match the true return-to-go.

**Insight**: Agents trained with dense reward produce smooth, accurate
Q-value trajectories.  Sparse-trained agents often have flat or noisy
Q-predictions because the value function was never shaped by intermediate
reward signal.

### Plot 4 – Credit Assignment Heatmap (episodes × timesteps)

A heatmap of |TD-error| for each (episode, timestep) pair under each
reward mode.  TD-error magnitude indicates where the Bellman backup is
most "surprised"—i.e., where credit assignment is hardest.

**Insight**: Dense reward distributes TD-error evenly; medium-delay
concentrates it at delivery boundaries; sparse shows a sharp spike only
at the terminal step, leaving the agent blind to intermediate performance.

### Plot 5 – Delayed Reward Robustness (delay steps vs performance)

The robustness sweep: x-axis is the delay length \(K\), y-axis is the
mean episode reward over the last 50 training episodes.  Error bars show
± 1 standard deviation.

**Insight**: Performance degrades roughly monotonically with delay.  A
critical threshold often exists beyond which DQN fails to learn meaningful
behavior, highlighting the practical limits of pure TD learning without
auxiliary reward shaping.

### Plot 6 – Learning Curve (training episode vs reward)

Smoothed training reward (original) over 500 episodes for all five
configurations.  Shows convergence speed and final performance.

**Insight**: Dense and baseline converge quickly to near-optimal
performance.  Medium-delay configurations learn more slowly but can still
converge given enough episodes.  Sparse reward typically fails to converge
within 500 episodes, underscoring the severity of the credit assignment
problem.


## 5. Key Findings

1. **Dense > Baseline ≥ Medium > Sparse** in convergence speed.  Extra
   shaping provides a mild but consistent advantage over the already-shaped
   LunarLander reward.

2. **Performance degrades super-linearly with delay.**  Doubling the delay
   window from 10 to 20 causes more than double the performance drop,
   consistent with the exponential decay of eligibility traces in TD(λ).

3. **Sparse reward effectively prevents DQN from learning** within 500
   episodes.  Without any intermediate signal, the replay buffer is
   dominated by zero-reward transitions and the Q-network has no gradient
   to propagate useful information backward.

4. **TD-error concentration** follows the delivery pattern precisely:
   credit assignment is *local to when rewards appear*, not when the
   underlying causal actions occurred.

5. **Value predictions under sparse reward are nearly flat**, confirming
   that the network fails to differentiate states by expected return.


## 6. Practical Implications

- **Reward shaping matters.**  Even mild additional shaping (the "dense"
  configuration) accelerates learning meaningfully.  Practitioners should
  invest in dense, informative reward functions.

- **For delayed-reward domains**, standard DQN is insufficient.
  Algorithmic extensions such as **n-step returns**, **hindsight experience
  replay (HER)**, **reward redistribution**, or **curiosity-driven
  exploration** are needed to bridge the temporal gap.

- **The robustness curve** (Plot 5) can serve as a diagnostic tool: if an
  agent's performance collapses at low delay values, the algorithm's
  credit assignment mechanism is weak and should be strengthened.

# key Findings from the Visualizations
Learning Curve (Plot 6): Dense and Baseline converge toward the solved threshold (200), while Medium Delay learns slowly and Sparse completely fails -- a clear ordering by reward information density.

Robustness Curve (Plot 5): Performance degrades monotonically from delay=1 (-30) through delay=20 (-450), then plateaus. The critical threshold is around K=15-20 steps.

Reward Timeline (Plot 1): Beautifully illustrates the sparsity pattern -- Baseline/Dense show per-step signals; Medium shows periodic bursts; Sparse collapses everything to a single terminal spike.

Return Variance (Plot 2): Sparse reward shows dramatically higher variance (~58,000) at early timesteps compared to Baseline/Dense (~10,000), confirming the credit assignment uncertainty.

Value Prediction (Plot 3): Baseline and Dense agents learn smooth, high Q-value trajectories (peaking at ~60). Sparse-trained agents show negative, erratic Q-predictions, indicating failed value learning.

Credit Assignment Heatmap (Plot 4): TD-errors concentrate at episode termination for all delayed configs, while Baseline distributes credit more evenly across timesteps.


可视化分析：

起因（图 1 - 奖励时间轴）： 物理上，奖励信号被截断和打包。本该每步都有的反馈，变成了隔很久才发放一次的脉冲，或者只在死前给算一次总账（Sparse）。

认知迷茫（图 2 - 回报方差）： 因为得不到及时的反馈，智能体对“我到底能拿多少分”感到极度迷茫。延迟越长，它在飞行初期的不确定性（方差）就越大。

远见丧失（图 3 - 价值预测）： 这种迷茫直接导致 Q 网络（对未来价值的预测）发生坍塌。没有每一步的真实奖励来锚定，贝尔曼方程无法将价值长距离回传，导致模型对好状态和坏状态的评分变得极其扁平和随机。

病灶所在（图 4 - 信用分配热力图）： 这是最致命的一击。因为奖励打包发放，网络在平时完全没有更新的动力（TD-Error 为 0），而在发奖励的那一瞬间，误差如海啸般集中爆发。这种极其不平滑的梯度直接摧毁了神经网络的稳定性。

能力边界（图 5 - 鲁棒性曲线）： 宏观上，这种微观的崩溃导致了性能的断崖式下跌。图表精准地测出了 DQN 在这个任务上的忍耐极限——大约 15 到 20 步的延迟。

最终结局（图 6 - 学习曲线）： 反映在最终的训练结果上，就是 Dense 和 Baseline 稳步通关；Medium Delay 苦苦挣扎、难以收敛；而 Sparse 彻底发散，沦为随机乱飞。
